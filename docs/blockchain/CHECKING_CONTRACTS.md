# 📋 How to Check Existing Contracts on Blockchain

This guide shows you how to view and verify contracts stored on your blockchain.

## Prerequisites

1. **Hardhat node must be running** (in a separate terminal):
   ```bash
   npm run hardhat:node
   ```

2. **Contract must be deployed**:
   ```bash
   cd blockchain
   npx hardhat run scripts/deploy.cjs --network localhost
   ```

3. **Update `.env`** with the deployed contract address

---

## 🔍 Check Blockchain Connection

Quick test to verify everything is working:

```bash
npm run blockchain:check
```

**What it checks:**
- ✅ Can connect to blockchain
- ✅ Contract is deployed
- ✅ Can interact with the contract
- ✅ Shows current block number and contract owner

## 📋 List ALL Contracts

View all contracts stored on the blockchain:

```bash
npm run blockchain:list
```

**Shows for each contract:**
- Contract ID
- Property ID
- Landlord & Tenant addresses
- Monthly rent & security deposit
- Start & end dates
- Contract status (Draft/Active/Expired/Terminated)
- Creation timestamp
- Transaction hash & block number

## 🔎 Get a Specific Contract

Look up details of a single contract by ID:

```bash
npm run blockchain:get YOUR_CONTRACT_ID
```

**Example:**
```bash
npm run blockchain:get 694406c5e3507dc54a9248b2
```

**Shows:**
- All contract details
- Verification that it exists on blockchain
- Current status

## 🔧 Troubleshooting

### ❌ "Can't connect to blockchain!"
**Solution:** Make sure Hardhat node is running:
```bash
npm run hardhat:node
```

### ❌ "NO CONTRACT at this address!"
**Solution:** Deploy the contract:
```bash
cd blockchain
npx hardhat run scripts/deploy.cjs --network localhost
```
Then update `RENTAL_CONTRACT_ADDRESS` in `.env` with the new address.

### ❌ "Contract does not exist"
**Possible reasons:**
1. The contract ID is incorrect
2. The contract was created in MongoDB but failed to save on blockchain
3. You're connected to a different blockchain (e.g., restarted the node)

**Solution:** Check your app logs when creating contracts for any blockchain errors.

### ❌ "Total Contracts: 0"
This means the smart contract is deployed but no rental contracts have been created yet. Create a contract through your app!

## 📊 Understanding the Data

### Contract Addresses
- **Landlord/Tenant addresses**: These are generated by hashing the user IDs from your app
- They are **deterministic** - same user ID always gives the same blockchain address
- This allows your app users to interact with blockchain without having real crypto wallets

### Amounts (Rent/Deposit)
- Stored in **Wei** (smallest Ethereum unit)
- Displayed in **ETH** for readability
- 1 ETH = 1,000,000,000,000,000,000 Wei

### Dates
- Stored as **Unix timestamps** (seconds since 1970)
- Displayed in human-readable format

### Status Values
- **0** = Draft
- **1** = Active
- **2** = Expired
- **3** = Terminated

### Terms Hash
- A **cryptographic hash** (SHA-256) of the contract terms
- Ensures terms cannot be changed without detection
- Used for verification

## 🔐 Contract Verification

To verify a contract's authenticity, you can:

1. **Check it exists on blockchain:**
   ```bash
   npm run blockchain:get CONTRACT_ID
   ```

2. **Compare the blockchain data** with your MongoDB data

3. **Verify the transaction hash** on the blockchain explorer (if using a public network)

4. **Check the terms hash** matches the original terms:
   ```javascript
   // In your app
   const crypto = require('crypto');
   const termsHash = '0x' + crypto.createHash('sha256')
     .update(JSON.stringify(terms))
     .digest('hex');
   ```

## 💡 Tips

1. **Keep Hardhat node running** - If it restarts, you'll lose all blockchain data
2. **Save important contract addresses** - Keep a record of deployed contract addresses
3. **Check regularly** - Run `npm run blockchain:list` to monitor your blockchain
4. **Test blockchain connection** before creating contracts in your app
5. **Check app logs** when creating contracts - they show blockchain transaction hashes
